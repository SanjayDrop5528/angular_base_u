shield Permission Service & Directives

The permission system uses PermissionService to check if users have specific permissions. Three directives are available:

  • [hasPermission]="permission" — Show/hide element based on permission
  • *hasAnyPermission="[p1, p2]" — Show if user has ANY of the permissions
  • *hasAllPermissions="[p1, p2]" — Show only if user has ALL permissions

dynamic_form Mode 3 — Component as Input

Pass any Angular component class to [cardComponent] on core-grid. The grid creates one instance per data row via ViewContainerRef, injects data and cardSchema into each instance, and appends it to the DOM. Your component just needs to implement the CoreCardHost interface.

Full component encapsulation Dynamic creation via ViewContainerRef Implement CoreCardHost interface
code TypeScript + Template
{{ snippet_mode3 }}

CoreCardHost interface

export interface CoreCardHost {
  data: any;
  cardSchema?: CardSchema;
  actionCallback?: (event: { action: string; data: any }) => void;
}

// In your component:
export class MyUserCardComponent implements CoreCardHost {
  @Input() data: any;
  @Input() cardSchema?: CardSchema;
  @Input() actionCallback?: (e: any) => void;
}

core-grid sets these inputs automatically after creating each instance. You can add any additional @Input() fields and populate them via a custom componentInputMap if needed.

grid_view core-grid — Adaptive Table + Card View

core-grid wraps AG Grid and automatically switches to card layout on mobile screens (< 768 px) via BreakpointObserver. The header provides manual toggle buttons (table / grid icons). You can force a specific mode with [viewMode]="'card'".

Input Type Description

(rowClicked) EventEmitter Emits the clicked row data
